7. 접근제한자와 static
(1) 접근 제한자: 
         클래스의 데이터와 메소드에 대해서 다른 클래스로부터 접근하는 제한선을 정한다.

(2) 접근제한자의 분류
1) public : 모든 클래스에서 사용 가능
2) protected:  해당 클래스와 동일한 패키지 안에서, 혹은 상속받은 클래스일 경우 접근 가능
3) default: 접근제한자를 명시하지 않은 경우이며, 같은 패키지 내에서 접근 가능.
4) private: 외부 객체 접근 불가 (해당 클래스 안에서만 사용 가능)

(2) static 변수 ( =class 변수, 정적 변수, 공용 변수) 의 특징 
- 클래스로부터 생성된 객체들의 수와 상관없이 하나만 생성된다. 
- 객체들끼리 공유하고 싶은 정보가 있을 때 사용한다. 
- 객체 생성과 상관없이 사용할 수 있는 필드와 메소드에 활용할 수 있다. 
- 프로젝트가 커지면 시스템 운영속도가 느려질 수도 있다는 단점이 있다.
